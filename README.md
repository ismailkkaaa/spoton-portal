# SpotOn File Portal Backend

Flask backend for the internal SpotOn File Portal.

## Features

- Simple session-based login for `Admin` and `Staff`
- SQLite-backed student and file metadata
- Country/course filtered student listing
- Per-student upload folders under `uploads/{student_id}/`
- File upload, download, and admin-only delete
- Clear JSON error responses

## Default Users

- `admin` / `admin123`
- `staff` / `staff123`

## Setup

```powershell
python -m venv venv
venv\Scripts\python -m pip install -r requirements.txt
venv\Scripts\python run.py
```

## API Overview

- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/auth/me`
- `POST /api/students`
- `GET /api/students?country=Georgia&course=MBBS`
- `GET /api/students/<student_id>`
- `PUT /api/students/<student_id>`
- `PATCH /api/students/<student_id>/status`
- `POST /api/students/<student_id>/files`
- `GET /api/students/<student_id>/files`
- `GET /api/students/<student_id>/files/<file_id>/download`
- `DELETE /api/students/<student_id>/files/<file_id>`

## Notes

- Student IDs are generated by country prefix:
  - Georgia -> `GEO-1001`
  - Uzbekistan -> `UZB-1001`
  - Tajikistan -> `TAJ-1001`
- Allowed file types are configured in `spoton_portal/config.py`
